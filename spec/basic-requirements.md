# Базовые требования

Проект: Meetup Reservation (монорепозиторий).

## Ограничения в технологиях

- Backend: ASP.NET Core (.NET 10), PostgreSQL, схема БД через SQL-скрипты, не использовать entity framework.
- Frontend: React + TypeScript + Vite.

## Описание приложения

Сервис мероприятий — это веб-приложение для организации, регистрации и управления событиями, таким как конференции, вебинары, мастер-классы или концерты. Оно упрощает создание событий, сбор участников и контроль посещаемости, делая процесс прозрачным для организаторов и удобным для посетителей.

### Что это такое

Платформа работает как Eventbrite или TimePad в мини-формате: организаторы публикуют события с датой, местом, ценой билетов; посетители регистрируются, оплачивают и получают доступ. Система автоматизирует рассылки, чек-ин и отчёты. Идеально для конференции — все поймут аналогию с регистрацией на доклады.

### Функциональные требования

- Создание события: Организатор добавляет название, описание, дату/время, место (онлайн/оффлайн), вместимость, цену билетов (бесплатно/платно), фото/программу.
- Регистрация посетителей: Поиск событий, выбор билета, ввод данных (ФИО, email, телефон), оплата (карта/онлайн), подтверждение с QR-кодом.
- Управление: Список участников, чек-ин по QR, отмена/возврат средств, экспорт в Excel/PDF.
- Автоматика: Уведомления (email/SMS о регистрации, напоминаниях за день/час), ограничение по местам (закрытие регистрации при заполнении).
