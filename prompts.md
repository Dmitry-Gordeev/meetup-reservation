
### UI UX Pro Max — Применение

```markdown
Примени UI UX Pro Max к проекту @src/frontend.

Цель: обновить UI/UX до современного, консистентного, production-ready уровня без изменения бизнес-логики.

Сделай по шагам:
1) Проанализируй текущий фронтенд в @src/frontend (структура, дизайн-система, компоненты, стили, доступность, адаптивность).
2) Сгенерируй рекомендуемую дизайн-систему (цвета, типографика, spacing, states, компоненты, паттерны страниц) под тип продукта meetup reservation.
3) Дай короткий план работ с приоритетами (quick wins -> medium -> advanced).
4) Реализуй изменения итеративно прямо в коде @src/frontend:
   - сначала токены/базовые стили
   - затем общие UI-компоненты
   - затем ключевые экраны
5) Обеспечь:
   - a11y (focus states, contrast, keyboard nav)
   - responsive (mobile/tablet/desktop)
   - единый стиль кнопок, форм, карточек, модалок
   - аккуратные hover/transition состояния
6) После каждого этапа показывай:
   - что изменено
   - какие файлы затронуты
   - как проверить результат локально

Ограничения:
- Не ломай существующую логику и API-контракты.
- Не вноси изменения вне @src/frontend без явной необходимости.
- Если чего-то не хватает (например, стек/библиотека UI), сначала спроси уточнение.
```
